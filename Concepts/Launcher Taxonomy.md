# Launcher Taxonomy

A way to bucket the field. Categories overlap; placement is by *primary* intent.

## By design philosophy
| Category | Intent | Examples |
|---|---|---|
| Power-user / customization | Control every behaviour | [[Nova Launcher]], [[Action Launcher]], [[Total Launcher]] |
| Stock-like / clean | Pixel feel, light tweaks | [[Pixel Launcher]], [[Lawnchair]] |
| Minimalist / intentional | Reduce distraction | [[Niagara Launcher]], [[Olauncher]], [[Before Launcher]] |
| Search-first | Type to find | [[KISS Launcher]] |
| Info-dense | Glanceable data dashboard | [[AIO Launcher]] |
| Adaptive / smart | Auto-organise, auto-theme | [[Smart Launcher]] |
| Ecosystem-tied | Hook into a vendor's services | [[Microsoft Launcher]], [[OEM Launchers]] |

## By license / source
- **Open source (FOSS)**: [[Lawnchair]], [[KISS Launcher]], [[Olauncher]]
- **Closed source, freemium**: [[Nova Launcher]], [[Niagara Launcher]], [[Smart Launcher]], [[Action Launcher]]
- **Closed, free (vendor-funded)**: [[Microsoft Launcher]], [[Pixel Launcher]], OEM defaults

## A quick decision frame
- *"I want it exactly my way"* → [[Nova Launcher]] / [[Total Launcher]]
- *"I want to use my phone less"* → [[Olauncher]] / [[Niagara Launcher]]
- *"Keep it stock but a bit better"* → [[Lawnchair]]
- *"Show me everything at a glance"* → [[AIO Launcher]]
- *"Organise it for me"* → [[Smart Launcher]]
