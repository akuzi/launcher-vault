# Gestures as Input

The home screen is a large, mostly-empty touch surface. Power launchers treat it as **invisible UI**: gestures map to actions without consuming screen space with buttons.

## Common gesture surfaces
- **Swipe up** — open app drawer (near-universal).
- **Swipe down** — open notifications / search.
- **Double-tap** — sleep screen, open assistant, lock.
- **Two-finger swipe**, **pinch**, **swipe on dock**, **edge swipes** — bound to arbitrary actions in advanced launchers.

## Why gestures matter
- **Reclaim space** — no need for shortcut icons cluttering the grid.
- **Speed** — a learned gesture beats navigating to an icon (cf. spatial vs gesture retrieval).
- **One-handed** use — bottom-anchored gestures suit large phones; central to [[Niagara Launcher]].

## Trade-offs
- **Discoverability** — invisible actions are unguessable; high setup + memory cost.
- **Conflict** with system navigation gestures (Android's back/home/recents swipes) can cause misfires.
- Mostly a **power-user** feature; minimalist launchers use few, deliberate gestures.

## Who does it well
- [[Nova Launcher]] — extensive, fully remappable gesture map.
- [[Action Launcher]] — Quickbar, Shutters, Covers are gesture-driven concepts.
- [[Niagara Launcher]] — one-handed pop-out menu and scrub list.
- [[Smart Launcher]] — gesture bindings plus the gesture-friendly "flower" dock.
